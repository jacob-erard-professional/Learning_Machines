# Implementation Plan: Community Health Request System

**Branch**: `001-community-health-request-system` | **Date**: 2026-03-21 | **Spec**: `specs/001-community-health-request-system/spec.md`
**Input**: Feature specification from `/specs/001-community-health-request-system/spec.md`

## Summary

Build a full-stack web application (React/Vite + Node/Express) that modernizes
Intermountain Healthcare's Community Health ordering and event support workflow.
The system accepts structured event/material requests through a web form, uses
the Claude API to auto-tag and classify requests via NLP, applies deterministic
routing logic (staff deployment vs. mail based on zip code service area lookup),
and provides an admin dashboard for search, editing, approval, and geographic
equity reporting. All routing decisions are admin-overridable. Approved staffed
events generate downloadable iCal invites.

## Technical Context

**Language/Version**: Node.js 20 LTS (backend), React 18 + Vite 5 (frontend)
**Primary Dependencies**: Express 4, React Router 6, Tailwind CSS 3, Zustand,
  Anthropic SDK (`@anthropic-ai/sdk`), `ics` (calendar), Vitest, Supertest,
  React Testing Library
**Storage**: In-memory JavaScript array (module-level); swap target is `lowdb`
  if persistence between demo sessions is needed. Render free tier has ephemeral
  FS вЂ” in-memory is the correct choice for the hackathon demo context.
**Testing**: Vitest + React Testing Library + jsdom (frontend);
  Vitest + Supertest (backend)
**Target Platform**: Web (desktop + mobile), deployed on Render free tier
**Project Type**: Full-stack web application
**Performance Goals**: Form submission + AI tagging response < 3s; page load < 2s;
  dashboard search/filter < 500ms (all in-memory)
**Constraints**: 6-hour hackathon build; Render free tier (ephemeral FS, cold starts);
  no external database setup; ANTHROPIC_API_KEY required for AI features
**Scale/Scope**: ~5 pages, ~8 API endpoints, ~10 components, hackathon demo audience
